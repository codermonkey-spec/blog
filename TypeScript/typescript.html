    # Typescript

    TypeScript 是具有类型语法的 JavaScript。

    TypeScript 是一种基于 JavaScript 的强类型编程语言，可为您提供任何规模的更好的工具。

    • 可以发现代码中的潜在问题
    • 代码提示更加的友好
    • 代码语义更加的清晰易懂

    # Node环境

    # 编译

    tsc会先将我们的typescript代码编译成js代码，然后进行执行,只不过在编译成js代码之前会先进行类型校验。

    我们通过tsc来执行代码他会生成一个对应名字的.js文件,但是有很多时候我们是不想要生成那个文件的，那么我们就可以使用ts-node来帮助我们来执行代码。

    # 通过ts-node来帮助我们编译代码

    // 全局安装ts-node
npm i ts-node -g

// 通过ts-node来编译
ts-node index.ts

    # null,undefined,void和never

    提醒

    在typescript中null和undefined类型比较特殊,他们都只有一个值

    null类型只有一个值为null,undefined类型也只有一个值为undefined

    undefined表示的是尚未定义,null表示的是缺少值

    提醒

    void是函数没有显示返回任何值时的返回类型,而never是函数根本不返回时使用的类型

    // 返回一个never的函数
function foo() {
  throw TypeError('error')
}

    # any类型

    任意值Any用来表示可以接受任何类型的值。

    当我们申明一个变量但是没有赋值的时候，ts默认会认为他是一个any类型

    let a;  // a is any

    # 类型注解和类型推断

    我们通过:类型来显示表示某一个变量是某一种类型的值,下面就是一个简单的注解

    let str:string = 'hello typescript'
    1

    那么类型推断又是什么呢？当我们明确知道一个变量是某一个类型的时候，我们可以使用类型注解。当我们不确定某一个变量是设么类型的时候，我们就可以让ts来帮助我们推导出对应的类型

    let name = 123; // ts会帮助我们推导出name为number类型
    1

    # 联合类型

    联合类型：表示取值可以为多种类型中的一种，多种类型使用|分隔开。

    let value: string | number
value = 123
value = '123'
    1
    2
    3

    WARNING

    当我们使用联合类型的时候，因为TypeScript不确定到底是哪一个类型，所以我们只能访问此联合类型的所有类型公用的属性和方法。



     


    // 这样的代码会报错，因为number类型的值是没有length属性的
function foo(arg:number | string) {
  return arg.length // error
}
